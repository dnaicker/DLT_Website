How it works.

  • An SSI mobile and web application was created to demonstrate capabilities through the interaction use case of an issuer and wallet owner connecting and verifying credentials sent. The framework used to create this dashboard is Trinsic SDK which is a tool that provides a layer of access to Hyperledger Aries and Hyperledger Indy to the Sovrin Staging network. Hyperleger tools are developed by IBM and are opensource tools to demonstrate SSI capabilities. Hyperledger Aries provides the network protocol layer of communication between different devices. Hypereldger Indy represent the distributed ledger on which credentials and policies are stored. Sovrin is an international non-profit foundation that contributes towards the development of these opensource initiatives in its mandate to create a global Sovrin identity network.

    There are two main users that form part of the system, an issuer and a wallet owner:

    1. An issuer is able to create credentials to send to wallet holders and verify whether those credentials being held are accurate via ZKP
    2. A wallet holder is able to store credentials sent through from an issuer as well as respond to verification requests from an issuer

    A third system entity that isnâ€™t involved in day-to-day interactions, is a provider that symbolically represents an organisation. Wallet holders and issuers conduct actions against each other using the API key that the provider provides them.

    The functionality presented from Issuer side:

    1. Create connection invitation in the form of a QR code or URL
    2. Create credential in this use case a login credential
    3. Verify a credential that a wallet holder might utilsing connectionId and walletId or through QR code which is represented as a connectionless verification check

    The functionality presented from Wallet owner side:

    1. Accept connection invitation from issuer
    2. Accept credential generated from issuer
    3. Respond to verification request to confirm credential sent from issuer is correct

Why it's important?

  • Identity management on the World Wide Web consists of creating a new username and password for accessing online services or utilising a centralised login through either Google Authentication or Facebook Login both of which share confidential information to third-party companies. To circumnavigate the shortcomings of these technologies SSI (Self-Sovereign Identity) provides a means of authenticating and certifying user requests through ZKP (Zero-Knowledge-Proofs) thus ensuring user privacy is upheld and data shared cannot be compromised by third-party companies.

SSI - Information

The need for SSI (Self-Sovereign Identity) in the World Wide Web is to introduce forms of identity management that obscure personal data transmitted and housed between online services to reduce the amount of illicit user tracking for profitable gain and mishandling of personal data that would compromise a persons safety in the physical world. The immutability of the distributed ledger provides a layer of authenticity from which one can issue uniquely generated identifiers for credentials and wallets. The efforts in the SSI domain is to utilise forms of digital credentials that are certified by authorised issuers which can be in the form of organisations, corporations or government institutions. To store these credentials on the user side, is done through a secure digital mobile wallet application or a desktop application that would be assigned a unique wallet identity number. This number is also stored on the distributed ledger, which will be then be used to assign digital credentials to. Verification of these credentials are done through a mechanism called Zero Knowledge Proofs (ZKP) which relies upon NP-completeness algorithms and attributes derived from credential verification policies to identify only the structure of the credential without compromising user confidentiality through extracting the value behind the attribute.
